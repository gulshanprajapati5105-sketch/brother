Understanding the Basics

What is Industrial IoT?

Industrial IoT (Internet of Things) connects your factory machines to a central system through the internet. It lets you see, understand, and improve everything happening on your factory floor — in real time, from anywhere.

Connect

Your machines talk to a central system. Every CNC, VMC, and Press machine sends data about what it's doing, right now.

Monitor

See everything on a live dashboard — which machine is running, idle, or stopped. Track temperature, power, speed, and production counts.

Optimize

Use the data to make better decisions — reduce downtime, save electricity, predict failures before they happen, and increase output.

The Building Blocks

What Hardware Do You Need?

Four key components sit between your machines and your dashboard. Here's what each one is, what it does, and why you need it.

1

Sensors

Attached directly to each machine

Small devices that physically measure what the machine is doing. They produce raw electrical signals — not readable data yet.

Current Sensor

Outputs 4-20 mA — tells if machine is ON/OFF and how hard it's working

Temperature Sensor

Outputs voltage — measures heat of spindle, motor, or coolant

Proximity Sensor

Outputs ON/OFF pulses — counts each job/part completed

Vibration Sensor

Detects abnormal shaking — early warning for mechanical failure

2

PLC / IO Module

Programmable Logic Controller — the "translator"

What is a PLC? A small industrial computer that reads raw electrical signals from sensors and converts them into meaningful data that software can understand.

What the PLC does (translation):

4 mA signal "Machine is OFF"
18 mA signal "Machine is RUNNING"
Pulse count "Jobs done: 47"
Voltage level "Temp: 62°C"
e.g. Siemens S7-1200 Allen-Bradley Modbus / OPC-UA output
3

Edge Gateway

Raspberry Pi / Industrial mini PC — the "packager"

A small computer sitting on the factory floor. It reads data from the PLC every 1-5 seconds and packages it into internet-friendly format (JSON) to send to the cloud.

What the gateway sends (every few seconds):

// Packaged as JSON →
{
  "machine": "CNC-01",
  "status": "RUNNING",
  "jobs": 47,
  "temp": 62,
  "power_kw": 4.2,
  "vibration": "normal"
}
Raspberry Pi 4 Sends via MQTT WiFi / Ethernet
4

MQTT Broker + Server

The "post office" + your backend application

MQTT Broker (like Mosquitto or HiveMQ) is a message router — the gateway publishes data to it, and your server subscribes to receive it. Think of it as a smart post office for machine data.

Your server then does 3 things:

Store — saves to Time-Series DB (InfluxDB / TimescaleDB)

Display — pushes live data to your dashboard via WebSocket

Alert — sends SMS/email if temp too high or machine stops

Node.js / Python REST API WebSocket

In short: Sensors (measure) → PLC (translate) → Gateway (package & send) → MQTT Broker (route) → Server (store, display, alert)

Under the Hood

Communication Layers

How does data travel from your machine to your screen? Here's each step explained simply.

1

Machine → Sensor

Smart sensors are attached to each machine. They read electrical signals, temperature, vibration, and other physical measurements directly from the machine's controller (PLC).

Modbus Protocol OPC-UA Protocol
2

Sensor → IoT Gateway

A small industrial computer (gateway) on the factory floor collects data from all sensors. It acts as the "brain" that gathers everything before sending it out. One gateway can handle 10-50+ machines.

Wired (RS-485) Industrial Ethernet
3

Gateway → Cloud Server

The gateway sends data to our secure cloud server over the internet. It uses MQTT — a lightweight messaging protocol designed for IoT. Data is encrypted and sent in real-time, every few seconds.

MQTT Protocol WiFi / 4G / Ethernet TLS Encrypted
4

Cloud Server → Processing & Storage

Our server receives the data, processes it (calculates uptime, detects anomalies, generates alerts), and stores it in a secure database. Historical data is kept for reporting and trend analysis.

REST API WebSocket Time-Series DB
5

Server → Live Dashboard (You)

You open a web browser on your phone, tablet, or computer and see everything live. Real-time updates, historical charts, alerts, and batch tracking — all in one beautiful dashboard.

Web Dashboard Mobile Friendly Real-Time
